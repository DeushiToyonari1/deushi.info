# プロジェクト開発ルール（出牛のブログ）

## 1. 開発環境の前提条件
- このプロジェクトは、MAMP上のWordPress（バックエンド）と、Vite + React（フロントエンド）を組み合わせた「ヘッドレスCMS」構成です。
- WordPressのベースURL: `http://localhost:8888/wp`
- フロントエンドの開発用URL: `http://localhost:5173`
- WordPressのAPI（`/wp-json`）や画像（`/wp-content`）は、Viteのプロキシ経由で配信されています。

## 2. コーディング規約と注意点
- **スタイルの修正:** CSS/レイアウトの追加や変更を行う場合は、他のファイルを汚さず、必ず `src/assets/styles/additional.scss` に追記してください。
- **レスポンシブ対応:** スマートフォン表示のブレイクポイントは `750px` を基準とします。メディアクエリは `@media (max-width: 750px)` を使用してください。
- **安全なコード変換:** WordPressのAPIから取得したメディアの絶対URLは、`src/api/client.ts` にある `normalizeWpUrls()` 関数を通して相対パスに変換し、Viteプロキシ経由で画像が正しく表示されるように配慮してください。

## 3. 画像最適化とSEOマークアップの自動化ルール
- **次世代フォーマットへの一括変換:** 記事で使用する画像（アイキャッチ画像や本文内の画像）を追加・処理する際は、ページの表示速度（Core Web Vitals）を最大化するため、オリジナルの画像に加えて、必ず高圧縮な「.avif」および「.webp」の2種類のフォーマットの画像を自律的に生成してアセットディレクトリに保存してください。
- **SEO効果の高いレスポンシブ・マークアップ:** フロントエンド（React）で画像を出力する際は、単なる `<img>` タグではなく、ブラウザの対応状況に応じて最適なフォーマットを自動で出し分けられるよう、以下の `<picture>` タグを用いた構造を徹底してください。
  ```html
  <picture>
    <source srcSet="[画像パス].avif" type="image/avif" />
    <source srcSet="[画像パス].webp" type="image/webp" />
    <img src="[オリジナルの画像パス]" alt="[適切な代替テキスト]" loading="lazy" />
  </picture>
  ```

## 4. タスク完了時の必須チェック
- コードの追加や変更を行った後は、必ずターミナルで以下のコマンドを実行し、TypeScriptのエラー（型エラー等）が発生していないか自律的に検証・確認してください。
  `npx tsc --noEmit`

## 5. バージョン管理（Git/GitHub）のルール
- すべての修正が完了し、型エラーがないことを確認したら、変更内容を必ず Git のステージ（git add）に追加してください。
- 変更内容に応じた適切なコミットメッセージ（Prefixに feat: や fix: を使用）を作成してコミットし、リモートリポジトリ（GitHubの master ブランチ）へのプッシュまでをタスクの最終工程として自動で実行してください。
